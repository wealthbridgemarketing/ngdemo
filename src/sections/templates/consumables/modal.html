AngularJS Dynamic Templates

What makes this application special is that the content was not present when the DOM was first parsed by Angular. The templates for each media type were in a separate json file, as was the content used to populate those templates.

The external files were loaded prior to the instantiation of the controller using the resolve property of this state in ui-router. This ensures that everything the app needs is available at runtime.

The LoadExternal service that I created to retrieve the files utilizes the LocalStorageService to cache the file contents. I've extended that service to support cache expiration and each file is cached for a different length of time.

The view is created using a custom directive and the ng-repeat attribute. 
<content-item ng-repeat="item in content" content="item"></content-item>
The directive does the following to attach the template and bind it to the content.
  • The content attribute in the directive passes the item to the directive.
  • The directive reads the content-type property of the item and retrieves the template for that type.
  • The template is added to the DOM.
  • The item properties are bound to the template using compile.

This technique was adapted from the following article:
http://onehungrymind.com/angularjs-dynamic-templates/
